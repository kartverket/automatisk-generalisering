Summary:

Selects hospitals and churches from the input point feature class, creating separate feature layers for each category.

Details:

  • Hospitals are selected based on ‘BYGGTYP_NBR’ values 970 and 719.

  • Churches are selected based on ‘BYGGTYP_NBR’ value 671.

generalization.n100.building.hospital_church_clusters.find_clusters()[source]
Summary:

Finds hospital and church clusters. A cluster is defined as two or more points that are closer together than 200 meters.

Details:

  • Clusters are found for both hospitals and churches using the ‘FindPointClusters’ tool.

  • The CLUSTER_IDs are joined with the original hospital and church feature classes.

  • Points belonging to a hospital or church cluster are selected as new layers.

  • Points not belonging to a cluster are selected as new layers.

Parameters:

Points'. (- The tool FindPointClusters has a search distance of '200 Meters' and a minimum of '2)

generalization.n100.building.hospital_church_clusters.reducing_clusters()[source]
Summary:

Reduces hospital and church clusters by keeping only one point for each detected cluster.

Details:

  • A minimum bounding polygon is created using the cluster points.

  • The same polygon is transformed into a centerpoint.

  • Only the cluster point nearest to the centerpoint is retained.

  • Hospital and church points not part of a cluster are merged with the selected cluster points.

Parameters:

AREA. (- Minimum Bounding Geometry is created with the geometry type RECTANGLE BY)

generalization.n100.building.simplify_polygons.simplify_buildings_1()[source]
Summary:

Simplifies building polygons to optimize for a 1:100,000 map scale. Building polygons that are under the minimum area value will be transformed to points.

Details:
  • The SimplifyBuilding tool is employed to reduce the complexity of building polygons, specifically tailored for the 1:100,000 map scale.

    It is used to decrease the number of vertices in building polygons, aiming to maintain essential features, but removing details for a smoother representation at the 1:100,000 scale.

Parameters:

  • Meters` (- Minimum Area is `3200 Square) – Controls the distance at which vertices are removed during the simplification process.

  • Meters` – Specifies the minimum area a simplified building polygon should have.

  • **`10 (- Minimum Hole Size is) –

    Sets the minimum size for holes within the building polygons to be retained.

  • Meters`** (000 Square) – Sets the minimum size for holes within the building polygons to be retained.

generalization.n100.building.simplify_polygons.simplify_polygons()[source]
Summary:

Reduces the complexity of building polygons even more to optimize for a 1:100,000 map scale, where small details of buildings may be perrceived as background noise.

Details:

  • This function simplifies the input building polygons even more to enhance visualization at the 1:100,000 map scale.

It reduces the number of vertices. Auto-generated points from the simplified polygons are also saved separately.

Parameters:

  • `15` (- Tolerance is) – It controls the simplification distance.

  • Meters` (- Minimum Area is `3200 Square) – Specifies the minimum area a simplified building polygon should have.
